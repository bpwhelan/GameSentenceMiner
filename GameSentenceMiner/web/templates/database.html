Download Special GSM Yomitan Dictionary

A character name dictionary that auto updates based on your most recently played games.

Increase if you frequently switch games. Include characters from 1-999 of your most recently played games (default: 3)
If set to 1 game, playing Manosoba then switching to Kanon will replace Manosoba character names with Kanon names when updating the dictionary in yomitan.
You may want to do this to keep your dictionaries clean and not confuse names across games. Names are sourced from Jiten/VNDB/AniList.
No Spoilers: Photo, Name, Game only.
Minor Spoilers: Description, roles, relationships, information.
All Spoilers: Full character descriptions + plot points that may spoil.
📝 How to update the dictionary in Yomitan/JL/Migaku

For Yomitan:

  1. Open Yomitan settings
  2. Navigate to the "Dictionaries" tab
  3. Click the "Check for updates" button
  4. Click "Update all"

For JL:

  1. When importing the dictionary, tell JL to update it daily.
  2. If you play a new game today and want the dictionary to upate right now, go to the dictionaries tab and manually click Update on it.

For Migaku:

  1. Delete the dictionary in Migaku, redownload it from here and import. Migaku does not support updating dictionaries.

External Stats Management

×

Add reading stats from any source (ッツ Reader, Kindle, physical books, etc.). Characters and time will be merged into your overall statistics.

Tag for grouping entries. Use consistent names like "ttsu", "kindle", etc.

Import reading data from Mokuro's volume-data.json file. Page turn timestamps are analyzed to compute per-day reading stats for each volume.

Recommended to avoid duplicates when re-importing. Only removes entries with source "mokuro".

Batch Import API

Use this endpoint to programmatically import stats from any source. Scripts, browser extensions, or other tools can POST entries directly.

POST /api/import-stats
Request Body (JSON) 
{
  "entries": [
    {
      "date": "2025-03-08",
      "characters_read": 5000,
      "time_read_seconds": 3600,
      "source": "ttsu",
      "label": "My Book Title"
    }
  ],
  "clear_source": "ttsu"
}
Fields
Field Type Required Description
entries array Yes Array of stat entries to import
entries[].date string Yes Date in YYYY-MM-DD format
entries[].characters_read int No Characters read (default: 0)
entries[].time_read_seconds float No Reading time in seconds (default: 0)
entries[].source string Yes Source identifier (e.g. "ttsu", "kindle")
entries[].label string No Book/volume title or description
clear_source string No Delete all existing entries for this source before importing (prevents duplicates)
Example (curl) 
curl -X POST http://localhost:58000/api/import-stats \
  -H "Content-Type: application/json" \
  -d '{
    "entries": [
      {
        "date": "2025-03-08",
        "characters_read": 5000,
        "time_read_seconds": 3600,
        "source": "ttsu",
        "label": "Novel Title Vol.1"
      },
      {
        "date": "2025-03-09",
        "characters_read": 3200,
        "time_read_seconds": 2400,
        "source": "ttsu",
        "label": "Novel Title Vol.1"
      }
    ],
    "clear_source": "ttsu"
  }'
Tip: Set clear_source to your source name to avoid duplicates when re-importing. The API accepts up to 50,000 entries per request. Entries with both characters_read and time_read_seconds equal to 0 are silently skipped.
